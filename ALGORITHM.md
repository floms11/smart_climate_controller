# 📊 Алгоритм роботи Smart Climate Controller

## Параметри налаштування

Всі параметри налаштовуються через UI в розділі Configuration:

- `minor_hysteresis` = **0.5°C** - поріг для включення роботи (мінімальна різниця для реагування)
- `major_threshold` = **1.0°C** - поріг для вимикання (максимальне відхилення перед вимкненням)
- `minor_correction` = **5.0°C** - невелика корекція температури на кондиціонері
- `major_correction` = **10.0°C** - велика корекція температури на кондиціонері

## 🔥 Режим HEAT (Обігрів)

### Приклад: Цільова температура 22°C

| Поточна t°C | Різниця (temp_diff) | Дія | Температура на AC | Опис |
|-------------|---------------------|-----|-------------------|------|
| 24.5 | +2.5 | ❌ **ВИМКНУТИ** | OFF | Занадто тепло |
| 23.5 | +1.5 | ❌ **ВИМКНУТИ** | OFF | Занадто тепло |
| 23.1 | +1.1 | ❌ **ВИМКНУТИ** | OFF | >1.0°C - вимикаємо |
| 22.8 | +0.8 | ✅ Працює | 22°C | Норма - підтримка |
| 22.3 | +0.3 | ✅ Працює | 22°C | Норма - підтримка |
| 22.0 | 0.0 | ✅ Працює | 22°C | Ідеально |
| 21.7 | -0.3 | ✅ Працює | 22°C | Норма - підтримка |
| 21.4 | -0.6 | 🔶 **Мала корекція** | 27°C (22+5) | <-0.5°C - трохи холодно |
| 20.8 | -1.2 | 🔥 **Велика корекція** | 32°C (22+10, обмежено до 30°C) | <-1.0°C - дуже холодно |
| 19.5 | -2.5 | 🔥 **Велика корекція** | 30°C (обмежено max) | Дуже холодно |

### Логіка HEAT:

```
IF temp_diff > +1.0°C:
    → ВИМКНУТИ кондиціонер (дуже тепло)

ELSE IF temp_diff < -1.0°C:
    → Велика корекція: AC temp = target + 10°C (дуже холодно)

ELSE IF temp_diff < -0.5°C:
    → Мала корекція: AC temp = target + 5°C (трохи холодно)

ELSE (-0.5°C ≤ temp_diff ≤ +1.0°C):
    → Працює нормально: AC temp = target (комфортний діапазон)
```

**Діапазон роботи:** від -0.5°C до +1.0°C відносно цілі

| 21.2 | -0.8 | ✅ Працює | 22°C | Норма - підтримка |
| 21.7 | -0.3 | ✅ Працює | 22°C | Норма - підтримка |
| 22.0 | 0.0 | ✅ Працює | 22°C | Ідеально |
| 22.3 | +0.3 | ✅ Працює | 22°C | Норма - підтримка |
| 22.6 | +0.6 | 🔶 **Мала корекція** | 17°C (22-5) | >+0.5°C - трохи спекотно |

```
IF temp_diff < -1.0°C:
    → ВИМКНУТИ кондиціонер (дуже холодно)

    → Велика корекція: AC temp = target - 10°C (дуже спекотно)

ELSE IF temp_diff > +0.5°C:
    → Мала корекція: AC temp = target - 5°C (трохи спекотно)

    → Працює нормально: AC temp = target (комфортний діапазон)
```

**Діапазон роботи:** від +0.5°C до -1.0°C відносно цілі

## 🔄 Перемикання режимів (HEAT ↔ COOL)

При зміні фізичного режиму (наприклад, з HEAT на COOL або навпаки):

- Кондиціонери, які знаходяться в **комфортному діапазоні**, автоматично **вимикаються**
- Це мінімізує непотрібну роботу кондиціонерів при зміні режиму
- Працюють тільки ті кондиціонери, які дійсно потрібні (з великою різницею температур)

**Комфортний діапазон:**
- HEAT: від -0.5°C до +1.0°C
- COOL: від -1.0°C до +0.5°C

## 🎯 Практичні приклади

### Приклад 1: Режим COOL, Ціль 22°C

**Ситуація:** Поточна температура 20.6°C
- `temp_diff = 20.6 - 22.0 = -0.6°C`
- Перевірка: `-0.6 < -1.0°C`? → **НІ**
- Перевірка: `-0.6 > +0.5°C`? → **НІ**
- **Результат:** В комфортному діапазоні (-1.0°C до +0.5°C)
- **Дія:** Кондиціонер **працює**, встановлює 22°C
- ✅ Кондиціонер **НЕ вимикається**

### Приклад 2: Режим HEAT, Ціль 22°C

**Ситуація:** Поточна температура 23.2°C
- `temp_diff = 23.2 - 22.0 = +1.2°C`
- Перевірка: `+1.2 > +1.0°C`? → **ТАК**
- **Результат:** Занадто тепло
- **Дія:** Кондиціонер **ВИМИКАЄТЬСЯ**

### Приклад 3: Режим COOL, Ціль 22°C

**Ситуація:** Поточна температура 22.7°C
- `temp_diff = 22.7 - 22.0 = +0.7°C`
- Перевірка: `+0.7 < -1.0°C`? → **НІ**
- Перевірка: `+0.7 > +1.0°C`? → **НІ**
- Перевірка: `+0.7 > +0.5°C`? → **ТАК**
- **Результат:** Трохи спекотно - мала корекція
- **Дія:** Кондиціонер встановлює 17°C (22-5)

## ⚙️ Додаткові налаштування

### Інтервали захисту

- **Min mode switch interval** (за замовчуванням 600 сек = 10 хв)
  - Мінімальний час між перемиканнями HEAT ↔ COOL
  - Запобігає частим змінам режиму роботи

- **Min power switch interval** (за замовчуванням 60 сек = 1 хв)
  - Мінімальний час між вмиканням/вимиканням
  - Захищає компресор від частих запусків

### Температурні пороги AUTO режиму

- **Outdoor temp for heat-only** (за замовчуванням 10°C)
  - Нижче цієї температури - тільки обігрів

- **Outdoor temp for cool-only** (за замовчуванням 23°C)
  - Вище цієї температури - тільки охолодження

- **Mode switch temperature threshold** (за замовчуванням 1.5°C)
  - Мінімальне відхилення для перемикання режимів в перехідній зоні

## 📝 Формули

### Різниця температури
```
temp_diff = поточна_температура - цільова_температура
```

### Температура на кондиціонері (HEAT)
```
IF temp_diff < -1.0°C:
    AC_temp = min(target + 10°C, 30°C)  # Велика корекція
ELSE IF temp_diff < -0.5°C:
    AC_temp = min(target + 5°C, 30°C)   # Мала корекція
ELSE:
    AC_temp = target                    # Норма
```

### Температура на кондиціонері (COOL)
```
IF temp_diff > +1.0°C:
    AC_temp = max(target - 10°C, 16°C)  # Велика корекція
ELSE IF temp_diff > +0.5°C:
    AC_temp = max(target - 5°C, 16°C)   # Мала корекція
ELSE:
    AC_temp = target                    # Норма
```

### Обмеження температури
- **Мінімум:** 16°C
- **Максимум:** 30°C
