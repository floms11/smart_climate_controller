# Швидкий старт

## Проблема: Кондиціонер не вмикається

**Симптом:** Ви змінюєте температуру на термостаті, але кондиціонер не реагує.

**Причина:** Термостат знаходиться в режимі OFF.

**Рішення:** Спочатку перемкніть термостат в один з режимів:
- **HEAT** - обігрів (кондиціонер буде працювати тільки в режимі heat)
- **COOL** - охолодження (кондиціонер буде працювати тільки в режимі cool)
- **AUTO** - автоматичний вибір режиму на основі температури на вулиці

## Як працює інтеграція

### 1. Термостат vs Кондиціонер

**Важливо розуміти:**
- **Термостат** (створюється інтеграцією) - це логічний контролер, який ви налаштовуєте
- **Кондиціонер** - це фізичний пристрій, яким керує термостат

### 2. Режими термостату

- **OFF** - термостат вимкнений, кондиціонер не працює
- **HEAT** - режим обігріву (кондиціонер працює в heat)
- **COOL** - режим охолодження (кондиціонер працює в cool)
- **AUTO** - автоматичний вибір heat/cool на основі температури на вулиці

### 3. Послідовність дій

1. **Виберіть режим термостату** (HEAT/COOL/AUTO)
2. **Встановіть цільову температуру** (наприклад, 22°C)
3. **Зачекайте 30 секунд** (update цикл coordinator)
4. **Кондиціонер почне працювати** з скоректованою температурою

## Як працює AUTO режим

### Пороги температури (налаштовуються в Options)

- **Outdoor temp for heat-only** (default 10°C) - нижче цієї температури доступний тільки режим HEAT
- **Outdoor temp for cool-only** (default 23°C) - вище цієї температури доступний тільки режим COOL
- Між цими порогами - **перехідна зона**, система сама обирає потрібний режим

### Логіка вибору в перехідній зоні

Система аналізує потреби всіх кімнат в групі:
- Якщо більшості потрібен обігрів → HEAT
- Якщо більшості потрібне охолодження → COOL
- При рівних потребах → зберігається поточний режим

## Корекція температури

Інтеграція не просто вмикає/вимикає кондиціонер, а **керує температурою на кондиціонері**.

### Приклад для режиму HEAT:

**Цільова температура:** 22°C

| Поточна температура | Що встановлюється на кондиціонері |
|---------------------|-----------------------------------|
| 20°C (холодно -2°C) | 32°C (target + 10°C) - сильна корекція |
| 21°C (холодно -1°C) | 32°C (target + 10°C) - сильна корекція |
| 21.3°C (холодно -0.7°C) | 27°C (target + 5°C) - дрібна корекція |
| 21.8°C (близько до цільової) | 22°C (target) - норма |
| 23°C (тепліше +1°C) | Кондиціонер вимикається |

### Приклад для режиму COOL:

**Цільова температура:** 22°C

| Поточна температура | Що встановлюється на кондиціонері |
|---------------------|-----------------------------------|
| 24°C (спекотно +2°C) | 12°C (target - 10°C) - сильна корекція |
| 23°C (спекотно +1°C) | 12°C (target - 10°C) - сильна корекція |
| 22.7°C (спекотно +0.7°C) | 17°C (target - 5°C) - дрібна корекція |
| 22.2°C (близько до цільової) | 22°C (target) - норма |
| 21°C (холодніше -1°C) | Кондиціонер вимикається |

## Мультиспліт групи

### Що це

Якщо у вас кілька кімнат підключені до однієї зовнішньої системи (multi-split), вони мають працювати в одному фізичному режимі.

### Як працює синхронізація

1. **Ви перемикаєте один термостат** в групі на HEAT
2. **Автоматично всі інші активні термостати** групи переходять в HEAT
3. **Всі кондиціонери групи** працюють в режимі HEAT
4. **Кожен кондиціонер** контролює свою температуру окремо

### Приклад

Група "main" з 3 кімнат:
- Спальня: термостат в AUTO, цільова 22°C
- Вітальня: термостат в AUTO, цільова 23°C
- Кухня: термостат в OFF

**Якщо на вулиці 18°C (перехідна зона):**
- Спальня: поточна 20°C → потрібен HEAT
- Вітальня: поточна 23°C → норма
- Система обирає HEAT (є потреба в обігріві)
- Спальня: кондиціонер в HEAT, 32°C (сильна корекція)
- Вітальня: кондиціонер в HEAT, 23°C (норма)
- Кухня: кондиціонер вимкнений (термостат OFF)

## Захист від частих перемикань

### Інтервали (налаштовуються в Options)

- **Min mode switch interval** (default 600 сек = 10 хв) - мінімальний час між перемиканням heat ↔ cool
- **Min power switch interval** (default 300 сек = 5 хв) - мінімальний час між вмиканням/вимиканням

### Що це дає

- Кондиціонер не "смикається"
- Компресор захищений від частих перезапусків
- Стабільна робота системи

## Діагностика

### Логи

Увімкніть детальне логування для діагностики:

```yaml
# configuration.yaml
logger:
  default: info
  logs:
    custom_components.smart_climate_controller: debug
```

### Що шукати в логах

**При зміні режиму:**
```
Thermostat Living Room: mode changed from off to heat
```

**При зміні температури:**
```
Thermostat Living Room: target temperature changed from 22.0 to 24.0 (mode: heat)
```

**При роботі coordinator:**
```
Processing group _independent_Living Room with rooms: ['Living Room']
Group _independent_Living Room hvac mode: heat
```

**При керуванні кондиціонером:**
```
Room Living Room: setting climate to mode=heat, temp=27.0 (current: mode=off, temp=None)
Room Living Room: mode changed to heat
Room Living Room: temperature set to 27.0
```

### Атрибути термостата

Кожен термостат показує додаткові атрибути:
- `physical_hvac_mode` - фактичний режим кондиціонера
- `physical_temperature` - встановлена температура на кондиціонері
- `outdoor_temperature` - температура на вулиці
- `multi_split_group` - назва групи (якщо є)

## Типові проблеми

### 1. Кондиціонер не вмикається

**Перевірте:**
- Термостат не в OFF режимі
- Пройшло 30 секунд після зміни
- В логах є "setting climate to mode=..."
- Сутність кондиціонера правильна і доступна

### 2. Кондиціонер не перемикається між heat/cool

**Перевірте:**
- Минуло достатньо часу (default 10 хв)
- В логах є "skipping mode change (... sec since last, min ... sec)"
- Зменшіть `min_mode_switch_interval` в Options

### 3. Термостати в групі не синхронізуються

**Перевірте:**
- У всіх кімнат однакова назва групи (з урахуванням регістру)
- В логах є "Synchronizing room ... to mode ..."
- Термостати не в OFF режимі

### 4. Температура на кондиціонері дивна (32°C або 12°C)

**Це нормально!** Інтеграція використовує корекцію температури:
- +10°C або -10°C - сильна корекція
- +5°C або -5°C - дрібна корекція
- Це примушує кондиціонер працювати ефективніше

## Налаштування параметрів

Settings → Integrations → Smart Climate Controller → Configure → Global Settings

### Основні параметри

- **Outdoor temp for heat-only**: 10°C (нижче - тільки обігрів)
- **Outdoor temp for cool-only**: 23°C (вище - тільки охолодження)
- **Minor correction hysteresis**: 0.5°C (поріг для дрібної корекції)
- **Minor correction value**: 5°C (значення дрібної корекції)
- **Major correction value**: 10°C (значення сильної корекції)
- **Major deviation threshold**: 1°C (поріг для сильної корекції)
- **Min mode switch interval**: 600 сек (мін інтервал між heat/cool)
- **Min power switch interval**: 300 сек (мін інтервал вмикання/вимикання)

### Коли змінювати

- **Холодний клімат**: зменште "outdoor temp for heat-only" до 5°C
- **Спекотний клімат**: збільште "outdoor temp for cool-only" до 28°C
- **Агресивніша корекція**: збільште correction values до 15°C
- **М'якша корекція**: зменште correction values до 3°C
- **Частіше перемикання**: зменште intervals до 300/180 сек
- **Рідше перемикання**: збільште intervals до 900/600 сек
