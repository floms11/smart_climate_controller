# Smart Climate Controller

Кастомна інтеграція Home Assistant для розумного керування кліматом через кондиціонери з підтримкою мультиспліт систем.

## Структура інтеграції

```
custom_components/smart_climate_controller/
├── manifest.json          # Опис інтеграції
├── __init__.py           # Точка входу, setup/unload
├── const.py              # Константи та параметри
├── config_flow.py        # UI конфігурація (додавання кімнат, налаштування)
├── coordinator.py        # Центральна логіка керування
├── climate.py            # Climate сутності (термостати для кімнат)
├── strings.json          # UI тексти
└── translations/
    └── en.json           # Переклади
```

## Основні компоненти

### 1. Config Flow (config_flow.py)

**Що робить:**
- Дозволяє додавати кімнати через UI
- Для кожної кімнати налаштовується:
  - Назва кімнати
  - Сутність кондиціонера
  - Датчик температури в приміщенні
  - Датчик зовнішньої температури
  - Група мультиспліт (опціонально)
- Налаштування глобальних параметрів:
  - Пороги температури для heat-only/cool-only режимів
  - Значення корекцій температури
  - Інтервали перемикань

**Options Flow:**
- Дозволяє змінювати глобальні параметри без перезавантаження
- Зміна кімнат вимагає видалення та повторного додавання інтеграції

### 2. Coordinator (coordinator.py)

Центральний компонент з усією логікою. Оновлюється кожні 30 секунд.

**Основні методи:**

#### a) Синхронізація термостатів у групі (`_synchronize_group_modes`)
**Де знаходиться:** `coordinator.py:508-538`

**Логіка:**
- Коли один термостат у групі перемикається в heat/cool/auto, всі інші активні термостати групи синхронізуються з ним
- OFF режим не тригерить синхронізацію
- Зберігається timestamp останньої зміни

**Виклик:** При зміні режиму термостату через `set_room_hvac_mode`

#### b) Вибір фактичного режиму кондиціонерів (`_determine_physical_mode`)
**Де знаходиться:** `coordinator.py:216-261`

**Логіка:**
1. Якщо група в HEAT → фізичний режим HEAT
2. Якщо група в COOL → фізичний режим COOL
3. Якщо група в AUTO:
   - Температура на вулиці < heat_only_threshold → HEAT
   - Температура на вулиці > cool_only_threshold → COOL
   - Посередині → викликається `_determine_mode_in_transition_zone`

#### c) Вибір режиму в перехідній зоні (`_determine_mode_in_transition_zone`)
**Де знаходиться:** `coordinator.py:263-345`

**Логіка:**
1. Перевіряє мінімальний інтервал від останнього перемикання режиму
2. Якщо інтервал не минув → зберігає поточний режим
3. Якщо інтервал минув → аналізує потребу кімнат:
   - За великі відхилення (> major_deviation_threshold) нараховується 10 балів
   - За малі відхилення нараховується 5 балів
   - Вибирається режим з найбільшим скором
   - При рівних скорах → зберігається поточний режим

**Це забезпечує стабільність і мінімізує зайві перемикання.**

#### d) Корекція температури (`_control_room_temperature`)
**Де знаходиться:** `coordinator.py:347-422`

**Логіка для режиму HEAT:**
1. Температура вище цільової на > major_threshold → вимкнути кондиціонер
2. Температура нижче цільової на > major_threshold → встановити target + major_correction
3. Температура нижче цільової на > minor_hysteresis → встановити target + minor_correction
4. В межах норми → встановити target

**Логіка для режиму COOL:**
1. Температура нижче цільової на > major_threshold → вимкнути кондиціонер
2. Температура вище цільової на > major_threshold → встановити target - major_correction
3. Температура вище цільової на > minor_hysteresis → встановити target - minor_correction
4. В межах норми → встановити target

#### e) Обмеження частоти перемикань (`_control_room_climate`)
**Де знаходиться:** `coordinator.py:424-519`

**Логіка:**
1. Перед зміною режиму перевіряє `min_mode_switch_interval`
2. Перед вмиканням/вимиканням перевіряє `min_power_switch_interval`
3. Якщо інтервал не минув → пропускає команду
4. Зберігає timestamps останніх змін у `RoomState`

#### f) Відновлення стану (`_restore_state` / `_save_state`)
**Де знаходиться:** `coordinator.py:565-612`

**Що зберігається:**
- Цільова температура кожного термостату
- Режим HVAC кожного термостату
- Timestamp останньої зміни режиму
- Timestamp останнього вмикання/вимикання

**Формат:** JSON через `Store` Home Assistant

### 3. Climate Entity (climate.py)

**Що робить:**
- Створює термостат для кожної кімнати
- Підтримує режими: OFF, HEAT, COOL, AUTO
- Відображає:
  - Поточну температуру (з датчика кімнати)
  - Цільову температуру
  - Режим термостату
  - Додаткові атрибути:
    - Фізичний режим кондиціонера
    - Температура на вулиці
    - Сутності датчиків
    - Група мультиспліт

**Управління:**
- `async_set_hvac_mode` → викликає `coordinator.set_room_hvac_mode`
- `async_set_temperature` → викликає `coordinator.set_room_temperature`

## Ключові моменти архітектури

### Розділення понять
1. **Режим термостату** (логічний) - що хоче користувач: heat/cool/auto/off
2. **Фізичний режим кондиціонера** (фактичний) - в якому режимі працює AC: heat/cool/off

### Синхронізація в групах
- **Термостати** синхронізуються автоматично при зміні режиму одного з них
- **Кондиціонери** в групі завжди працюють в одному фізичному режимі
- Незалежні кімнати (без групи) не синхронізуються

### Стабільність роботи
1. Мінімальні інтервали перемикань запобігають "смиканню"
2. В перехідній зоні використовується scoring system для вибору режиму
3. При рівних потребах зберігається поточний режим
4. Стан зберігається після перезапуску

### Можливість розширення
- Легко додати нові параметри в `const.py`
- Логіка розділена на окремі методи
- Можна розширити на інші типи пристроїв
- Coordinator використовує стандартний паттерн Home Assistant

## Використання

### Встановлення
1. Скопіювати папку `smart_climate_controller` в `custom_components/`
2. Перезапустити Home Assistant
3. Додати інтеграцію через UI: Settings → Integrations → Add Integration → Smart Climate Controller

### Конфігурація
1. Додати кімнати (одну за одною)
2. Для кожної кімнати вказати:
   - Назву
   - Кондиціонер
   - Датчики температури
   - Групу (якщо є мультиспліт)
3. Налаштувати глобальні параметри

### Зміна параметрів
- Settings → Integrations → Smart Climate Controller → Configure
- Вибрати "Global Settings"
- Змінити потрібні параметри

## Діагностика

### Логи
Інтеграція пише детальні логи:
- Зміна режимів термостатів
- Синхронізація груп
- Вибір фізичного режиму
- Корекція температури
- Пропущені команди через інтервали

### Атрибути термостатів
Кожен термостат показує:
- `physical_hvac_mode` - фактичний режим кондиціонера
- `physical_temperature` - встановлена температура на кондиціонері
- `outdoor_temperature` - температура на вулиці
- `multi_split_group` - група (якщо є)

## Приклади сценаріїв

### Сценарій 1: Незалежна кімната
- Кімната без групи
- Термостат в AUTO
- Логіка працює тільки для цієї кімнати

### Сценарій 2: Мультиспліт система
- 3 кімнати в одній групі "main"
- Користувач перемикає кімнату 1 в HEAT
- Автоматично кімнати 2 і 3 переходять в HEAT
- Всі 3 кондиціонери працюють в режимі HEAT
- Кожен контролює свою температуру

### Сценарій 3: Перехідна зона
- Група в AUTO
- Температура на вулиці 18°C (між порогами 10°C і 23°C)
- Кімната 1: потрібно +2°C (скор heat = 5)
- Кімната 2: потрібно -0.5°C (скор cool = 5)
- Рівні скори → зберігається поточний режим
- Якщо в кімнаті 1 потрібно +2°C (скор heat = 10) → група переходить в HEAT

## Розробка

### Редагування логіки
Вся логіка в `coordinator.py`, розділена на методи:
- `_process_group` - головний цикл обробки групи
- `_determine_group_hvac_mode` - синхронізація термостатів
- `_determine_physical_mode` - вибір фізичного режиму
- `_determine_mode_in_transition_zone` - логіка перехідної зони
- `_control_room_temperature` - корекція температури
- `_control_room_climate` - виконання команд з перевіркою інтервалів

### Додавання нових параметрів
1. Додати константу в `const.py`
2. Додати в схему `config_flow.py` (initial setup та options)
3. Додати в `strings.json` та `translations/en.json`
4. Використовувати через `self._get_global_option(CONF_NEW_PARAM, DEFAULT_VALUE)`

### Тестування
- Тестувати на різних сценаріях температури
- Перевірити логи на наявність циклів перемикань
- Переконатись що стан відновлюється після перезапуску
